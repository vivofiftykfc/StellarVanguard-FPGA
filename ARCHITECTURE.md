# ARCHITECTURE — STELLAR VANGUARD 系统架构

## 1. 模块层次结构

```
top.v (顶层)
├── vga_ctrl.v           # VGA 时序生成 (640x480@60Hz)
├── game_engine.v        # 游戏逻辑核心
├── render.v             # 像素渲染管线
│   └── font_rom.v       # 8x8 字符 ROM
├── button_debounce (x4) # 按钮消抖 (top.v 内联子模块)
└── 7-seg MUX + BCD      # 数码管显示 (top.v 内联逻辑)
```

## 2. 时钟域

```
CLK100MHZ (100MHz, 片载晶振)
    │
    ├──> clk_div[1:0] 计数器
    │       │
    │       └──> vga_tick (25MHz 使能脉冲, 每 4 个 CLK100MHZ 一次)
    │              ├──> vga_ctrl (时序生成)
    │              └──> render (像素输出寄存器)
    │
    ├──> game_engine (100MHz, new_frame 使能)
    │       所有游戏逻辑在 new_frame 脉冲时更新 (60Hz)
    │
    └──> 7-seg MUX (100MHz 自由运行)
```

**关键设计决策**: 不使用 PLL/MMCM 生成 25MHz 时钟，改用 100MHz + 使能脉冲方式。好处：
- 整个设计只有一个时钟域 (100MHz)，无需跨时钟域处理
- VGA_ctrl 和 render 仅在 `vga_tick` 有效时推进
- 时序收敛简单（所有路径约束为 100MHz = 10ns）

## 4. 各模块详细设计

### 4.1 vga_ctrl.v — VGA 时序控制器

```
接口: clk(100MHz), tick(25MHz使能), rst_n
输出: hsync, vsync, h_count[9:0], v_count[9:0], active, new_frame

时序参数 (640x480@60Hz):
  行: 800 total = 96 sync + 48 back + 640 disp + 16 front
  场: 525 total = 2 sync + 33 back + 480 disp + 10 front

new_frame: tick && h_cnt==0 && v_cnt==0 (每帧一个脉冲)
```

### 4.2 game_engine.v — 游戏逻辑核心

```
子模块 (均为内部 always 块):
├── 状态机: TITLE / PLAYING / DYING / GAMEOVER
├── 玩家: 位置 + 生命 + 无敌计时 + 射击冷却
├── 星空: 32 颗星, LFSR 随机, 向下滚动
├── 敌机: 20 槽位, 3 类型, 波次管理
├── 子弹: 5 槽位, 玩家发射, 向上飞行
├── 碰撞检测: 子弹 vs 敌机 (AABB), 玩家 vs 敌机
├── 爆炸特效: 5 槽位, 曼哈顿距离, 生命周期
└── 计分: score_reg

关键设计参数:
  MAX_ENEMIES   = 20   (需 5 bits)
  MAX_BULLETS   = 5    (需 3 bits)
  MAX_EXPLOSIONS = 5   (需 3 bits)
  NUM_STARS     = 32   (需 6 bits! 5'd32 会溢出为 0)
```

### 4.3 render.v — 像素渲染管线

```
架构: 分层独立 always @(*) 块 + 最终优先级 MUX

第 1 层: 背景 (深蓝渐变)
第 2 层: 星空 (32 颗, 点比较)
第 3 层: 爆炸 (5 个, 曼哈顿距离 + 颜色渐变)
第 4 层: 敌机 (20 个, 按类型绘制不同形状)
第 5 层: 子弹 (5 个, 亮黄 + 中心高亮)
第 6 层: 玩家 (16x16 精灵)
第 7 层: HUD (SCORE/WAVE/LIVES + 生命图标)
第 8 层: 标题画面 (文字 + 装饰线 + 星星)
第 9 层: 游戏结束 (半透明遮罩 + 文字)
最终层: 状态选择 + 优先级 MUX + 文字 font ROM 查询

设计要点:
- 每层独立 always @(*) → Vivado 可并行综合各层
- 最终层只做选择/覆盖，不包含游戏对象渲染
- font_rom 在最终层查询（共享资源，避免每层复制）
- 输出寄存器化 (vga_tick 使能)
```

### 4.4 font_rom.v — 字符 ROM

```
接口: char_idx[5:0] (0-42), row[2:0] (0-7)
输出: pixels[7:0] (8 像素位图, MSB=左)

字符集 (43 字符):
  0-9: '0'-'9', 10-35: 'A'-'Z', 36-42: 空格 ! . : - > /

实现: 纯组合逻辑 case({char_idx, row}) → 推断为 LUT
```

### 4.5 top.v — 顶层集成

```
组件:
├── 时钟分频: 2-bit 计数器 → vga_tick (25MHz 使能)
├── 按钮消抖: button_debounce x4 (500us 稳定时间)
├── vga_ctrl 实例化
├── game_engine 实例化
├── render 实例化
├── 消隐处理: active ? pix : 黑色
├── 7-Seg BCD: double-dabble 组合逻辑
└── 7-Seg MUX: 多路复用扫描 (~610Hz)
```

## 5. 模块间接口 (展平 packed vector 规范)

所有数组信号必须展平为 packed vector:

| 信号 | 条目数 | 每条目宽度 | 总宽度 | 索引方式 |
|------|--------|-----------|--------|----------|
| enemy_x | 20 | 10 | [199:0] | `[i*10 +: 10]` |
| enemy_y | 20 | 9 | [179:0] | `[i*9 +: 9]` |
| enemy_type | 20 | 2 | [39:0] | `[i*2 +: 2]` |
| bullet_x | 5 | 10 | [49:0] | `[i*10 +: 10]` |
| bullet_y | 5 | 9 | [44:0] | `[i*9 +: 9]` |
| exp_x | 5 | 10 | [49:0] | `[i*10 +: 10]` |
| exp_y | 5 | 9 | [44:0] | `[i*9 +: 9]` |
| exp_life | 5 | 5 | [24:0] | `[i*5 +: 5]` |

## 6. 资源估算

| 模块 | LUT 估算 | FF 估算 | 备注 |
|------|---------|---------|------|
| vga_ctrl | ~50 | ~30 | 简单计数器 |
| game_engine | ~3000 | ~2000 | 碰撞检测 + 多槽位管理 |
| render | ~5000 | ~100 | 巨型组合逻辑 (拆分后) |
| font_rom | ~2800 | 0 | 纯组合 |
| top (7-seg等) | ~500 | ~200 | BCD + MUX |
| button_debounce | ~200 | ~300 | 4x 消抖 |
| **合计** | **~11,550** | **~2,630** | < 50% XC7A35T |
